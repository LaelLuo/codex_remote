# 自定义 Codex 可执行文件选择支持

## 目标

- 支持用户在配置中自定义选择启动和调用的 Codex 可执行文件路径或命令，而不是只能依赖固定默认可执行文件。

## 范围

### In

- 识别当前 Anchor 和相关启动链路里对 Codex 可执行文件的解析、启动与校验位置。
- 设计并实现可配置的 Codex 可执行文件来源，覆盖默认值、显式配置与兼容回退。
- 补齐 UI 或配置入口以及必要的验证与错误提示。

### Out

- 不改动 Codex Remote 的认证协议与 Orbit 控制面行为。

## 验收标准

- 用户可以显式配置一个自定义 Codex 可执行文件。
- 未配置时保持现有默认行为不回归。
- 配置无效时有可理解的报错或回退策略。
